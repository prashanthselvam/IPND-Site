Stage 0: Getting Started with HTML

The World Wide Web

The World Wide Web was invented in the early 90's. It is made up of webpages which are created using Hyper Text Markup Language or HTML. It sounds like a mouthful but is rather quite simple.

The web consists of three key components: Your web browser, the internet and the servers where all the data is stored. The web browser sends messages through the internet in HTTP. The server responds by sending the required files, images, webpages, etc. back through the internet to the web browser. This is how you see what you see when using the internet.

HTML

HTML is made up of four key components:

  • Text Content: What you see
  • Markup: What the content looks like
  • References to other documents: Images, videos, etc.
  • Links to other pages

Markup

Markup, or what the content looks like, is determined by various tags which we can use in HTML. These tags aren't visible to the user but they control the layout and design of the contents on the webpage. Below are a few examples of tags:

  • <b>Content</b> will make the content appear bold
  • <em>Content</em> will italicize the content

There are also attribute tags which are formatted as follows:

  • <Tag attr="value">contents</Tag>

An example of such an attribute tag is a href which is used to create hyperlnks

Whitespace

HTML does not recognize the whitespace that is included in the code. So we may enter as many spaces as we'd like between words in the HTML editor but it will be displayed as one space in the HTML page.

To create whitespace, there are a couple tags we can use. The <br> tag is one such example which is used to create breaks. This is the equivalent of pressing enter on a word document. Another tag is the <p> tag which creates paragraphs. There is a key difference between these two tags:

The <br> tag is an example of an inline tag while <p> is a block tag.The difference between the two will get clearer as we go but essentially an inline tag occurs within the line, while a block tag creates a space around the content. Block tags generally need to have a closing tag while inline tags don't always need to.

Stage 1: Make Your Webpage

The Website As A House

A website consists of many components which come together in a coherent way. As such, the website can be compared to a house.

  • HTML: The structure of the house; the walls/where everything is located
  • CSS: The design, the furniture, the color scheme
  • Javascript: The more interactive features like the garage door remote

All these elements come together in a tree-like structure through the DOM or Document Object Model. DOM is the convention of presenting and interacting with elements in the web-tree.

Boxes Everywhere!

Webpages can be seen as a group of boxes. If we take any webpage, we can break it down into a set of differently shaped boxes, each containing a different element. This makes it easy for us to design webpages. These boxes are created using <div> tags which will be explained in more depth soon.

CSS

CSS or Cascading Style Sheets is the language which is used to add style to the webpage. It's described as "Cascading" because the style rules which are applied keep going down many levels - from child to parent - until the most specific rule gets applied.

This relates to the idea of inheritance which is the mechanism by which properties are applied not only to a specified element but also its descendants. Generally speaking, text-related properties are inherited (font, size, color, etc.) but box-related properties (box-width, height, etc.) are not.

The Box Model

As mentioned earlier, elements on webpages are organized in boxes. Everything on the net, is boxes. These boxes are composed of four key parts and we can play with each:

  • Content
  • Padding
  • Border
  • Margin

For more info on the box model, you can refer to this tutorial.

Ways of Working in Programming

In order to be a succesful programmer, a basic way of working needs to be followed. This can be described as Code, Test, Refine:

  1. Get the basic structure of the page right
  2. Translate the structure into tags within HTML
  3. Start adjusting basic style - size of the boxes
  4. Adjust positioning of the boxes
  5. Once this general layout has been accomplished, dive into the details - colors, fonts, etc.

Avoiding Repetition

Another key part of being a good programmer is avoiding repetition. This not only makes life a lot easier, but it keeps the program well-structured and light. Reptition slows down everything - the programmer, the page, and any person in the future who might have to work with your code. Avoiding repeition is a key tenet of programming. For example, using CSS for particular div or span classes is an example of avoiding repetition. By assigning a particular attribute to that class, we don't have to repeat it each time it appears in the code.

Stage 2: Introduction to Serious Programming

Programming

Before we delve into programming, it is important to note a few definitions of key concepts. These are listed below:

  • Computer: A universal machine which can execute precisely defined steps incredibly rapidly.
  • Program: Something which tells the computer what to do. It is essentially a sequence of very precise steps
  • Programming Language: The language in which the above mentioned steps are written. The programming language we will be learning is Python, which is also an interpreter.

Anything from a simple print command in Python, to the calendar app on a phone, to Python itself is an example of a program. It runs on sets of very precise instructions.

Language Ambiguity

Some may ask why we use these complex languages for programming instead of the natural languages we already have. The reason is natural languages are inherently ambiguous and they are also too verbose. As mentioned earlier, the computer is stupid in that it can simply follow the instructions it is given (very rapidly). If these instructions are unclear, the computer cannot execute. Hence, natural language is not viable as a programming language.

Programming Grammar

Like any language, programming languages have their syntax and grammar, and same goes for Python. One of the standard grammatical structures of programming statements is the Backus Naur Form. This video provides an excellent explanation.

Stage 3: Python Concepts

Variables And Strings

A variable is an element in programming languages which can hold a certain value. As the name suggests, variables can vary. In other words, their value can change. Variables are one of the most fundamental building blocks of programming.

To assign a value to a variable, you follow the form "variable_name=value". Variables can contain numbers or strings.

There is a lot one can do with strings and variables. For example, we can index strings which is a way of identifying specific characters within the string. By using the in-built find function, we can extract the location of specific substrings from within a string.

Functions

Like variables, functions are another fundamental building block of programming languages. Much of what programmers do is creating and using functions which can do anything from very simple tasks to incredibly complex. A function is defined using the following code:

def function_name(function input):
        return: output

Functions are pretty much like the functions we have in mathematics. They take a certain input and transform it into a certain output. This has utility in an endless number of contexts.

The benefits of writing functions are numerous. For one, they allow us to break down a large program into smaller steps. This not only makes the code more coherent and well-organized, it also allows us to diagnose issues more easily by running each function in isolation. Functions also ensure that we don't repeat a lot of code, as the same function can be called to run the same code on different inputs. Another benefit is that local variables within the function are live only as long as the function is alive. So a variable x in function_1 will not interfere with a variable x in function_2. This avoid issues of variables being accidentally assigned values which were not intended for them. For more information on the benefits of function, please refer to the following link

If & While Statements

If and while statements are incredibly important in programming. Most programs, in fact, are a lot of if/while statements (and variations of if/while statements) which are placed within functions. An if statement is exactly what it sounds like: if a certain statement is true, perform this action. While is similar, but it creates a loop. So we are saying, while this statement is true, keep performing this action.

Debugging

Debugging seems like the dryest part of programming but it may be one of the most important skills you need to be a good programmer. It is essentially finding and fixing the errors in your program. In the lesson, 5 key strategies for debugging were covered:

  1. Examine error messages when programs crash
  2. Work from example code
  3. Make sure examples work
  4. Check (print) immediate results
  5. Keep & compare old versions

Each of these is covered in more detail in this lesson

Stage 4: Diving Deeper in Python

Structured Data

One of the most powerful data structures in python is the list. It can be defined as a sequence of any type of elements. A list is created as follows:

p = ['a','b','c']

Each element of the list can be a string, or a number, or even other lists. Lists are pretty great and there is a lot you can do with them because of their versatility.

List Operators

Like we have for strings, we have various operators we can use to manipulate lists. The most basic would be list indexing. To refer to a specific element in a list, we use the list_name[element_number] format, much like we did with strings. To manipulate the list, here are a couple operators:

Sum Operator: By adding two lists, ['a','b'] + ['c','d'] we simply concatenate them into ['a','b','c','d'].

Append Operator: list.append(element) will add to the list one more element. This element may be a string, a number, or even another list.

There is also the for statement which is effectively a while statement for lists. This is also incredibly useful in writing more efficient code.

How To Solve Problems

There is no one golden bullet to approaching every problem in the world. Each requires its own style. However, there are some general steps we can follow in programming which can make us a lot more adept at solving problems. These steps are really quite simple:

  1. Don't panic!
  2. What are the inputs
  3. What are the outputs
  4. How do I get from step 2 to 3

Granted, that last step is a lot harder than the other three. However, the broader message here is to tackle the problem systematically. Once we are on step 3, we can break the problem into smaller parts and try to resolve each. Being able to think in abstract terms also helps because you are able to see the structures upon which the problem's solution would be based.

Stage 5: Let's Make a Real Webpage

Intro to Networks

Networks can be found everywhere. They are essentially systems of various elements which can coherently communicate with each other. The internet is one massive network which consists of many subnetworks. Communication within the internet happens via servers. When you go to a certain URL, your client (the web browser) sends a request to a server which holds all the data which makes up the website you're trying to visit. Similarly, when you post something on the website, your web client sends a request to the server to post the data on to the site. Essentially, while get is used for fetching documents from the server, post is used for updating data on the server. Also, GET parameters live in the URL. Because of this, GET commands have limited lengths. POST commands, on the other hand, because the parameters are in the body of the document, can be almost limitless in length.

Also important to discuss is HTTP. HyperText Transfer Protocol is the main protocol of the web. It is what your browser uses to talk to servers. When you type a URL into your browser, your browser is actually sending a get request to the server. So say you are trying to visit the URL: http://www.example.com/foo the request line will be GET /foo HTTP1.1. There are three main parts in that line (The method, the path, and the version). The method is the method in which you are interacting with the server (GET and POST are the two big ones). The path is the actual document you are requesting from the server. The last bit is the version which is the version of HTTP that you are speaking. If you're wondering where the www.example.com is in this request line, it's not there because you already connected to it when the page loaded. www.example.com would be the root directory or the first "/" that you see before foo.

When a request like the one above is made, the server responds with a response statement. This respons can look like: HTTP/1.1 200 OK. The 200 in there is a status code. There are a few different status codes and their meanings are as follows:

  • 200 OK - The document is found
  • 302 Found - The document is located somewhere else
  • 404 Not Found - The document was not found at all (You've probably seen this one)
  • 500 Server Error - The server broke trying to process your request

These status codes allow us to debug our code and see what's going on between the server and the browser. The status line has some more elements which you can learn about in this video

Finally, let us look at the two broad types of responses the server can send - Static vs Dynamic. The names are pretty self explanatory but let us look into them deeper. Static is pre-written content that the server simply returns. Images, or most old webpages are pretty much static. Dynamic requests receive content that's made on the fly by web applications. Most content we see on the web today is dynamic, i.e. it has been created by web applications which live on servers. And these web applications are what we are learning to create :)

Validation

When creating websites, we often need to include forms which are used to take in user input. This is a hugely useful tool and is a big part of what makes websites dynamic and exciting. However, this also adds the risk of users being able to enter malicious or even well-intentioned invalid content to the site, which may mess up the code. This is why we need validation.

Validation is basically ensuring that whatever the user has inputted into our website is fit for the website to handle. This can be done in many different ways, but they all involve, in one way or another, using code to understand what the user has entered and seeing if it lines up with the type of value we were expecting. So, for example, if a user enters letters in a field where we need the user to input digits, then we can validate that with code such as isdigit().

Related to the concept of validation is escaping. When users enter html tags into forms, it has the capability of seriously messing up the page. This is because the user's HTML tags enter our website as a string and mess up the existing code. Making the system understand when the user is entering these tags and "escaping" them is crucial.

Validation also significantly improves the user experience on a website. With validation, we can do things like display error messages when a user inputs incorrect or invalid information in a form. On this website, for example, in the guestbook we have used validation to recognize if the user has entered information in both entry fields (name and content). If the server recognizes that either field is empty, it throws out an error message which is stored in the error variable. This allows the user to see why the website is not behaving the way they were expecting it to, and correct their mistakes. The importance of avoiding repetition

Avoiding repetition is a key tenet in writing good code. We have been avoiding repetition throughout this course - Using functions and variables, for example, are techniques which in a way avoid repetition. The reasons for avoiding repetition are fairly obvious - it saves you lots of time, it makes your code a lot neater, it makes your applications less bulky, and it is easier to correct errors and fix things in multiple locations all at once. CSS Frameworks and Responsive Layouts

A big part of making good design is having strong frameworks. Design frameworks are the basic structures within which the contents of the page are organized. If you look at almost any webpage, you'll notice you can organize its contents very easily into some sort of grid structure. This is the design framework of the website. It is not just important to have a strong framework, but also that it be responsive to different scenarios. A framework which adjusts to the medium on which it is being viewed is crucial to ensuring that users have a consistent experience on any platform.

We have already discussed some of the elements of frameworks building in the first few lessons (boxes, flex, etc.) Let's dive a little further into some of these concepts.
